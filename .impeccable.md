## Design Context

### Users
Players of the finished scoped game on desktop keyboard and phone touch, plus future maintainers extending the fighting-game engine. Players should be able to jump into a polished arcade fight quickly, read the current combat state at a glance, and feel that the game is smooth enough to be worth paying to try.

### Brand Personality
Playful, premium, kinetic. The interface should feel like a vibrant arcade cabinet for an original rabbit-versus-tabby fighting game: expressive, colorful, responsive, and confident without exposing debug or asset-pipeline scaffolding to players.

### Aesthetic Direction
Bright 2D arcade fighting-game presentation with a warm Meowtal Courtyard stage, generated character sprites, ornate HUD surfaces, sharp fight-state feedback, and intentional mobile framing. The visual language should stay original to Meowtal Kombat while using King of Fighters-style smoothness and Tough Love Arena-level browser polish as feel benchmarks.

### Design Principles
- Prioritize playable clarity: effects, framing, and UI should make fighters, health, timer, guard state, and touch controls easier to read.
- Preserve character consistency: the gray rabbit and ginger tabby should keep their approved upright two-legged silhouettes, proportions, and generated-art style across all presentation states.
- Treat mobile as a real arcade context: touch targets need comfortable scaled size, thumb-reachable placement, and an intentionally composed portrait/landscape frame.
- Keep the first screen and every state game-native: no generic landing-page chrome, debug labels, or asset-pipeline copy in player-facing views.
- Verify visual changes in a browser with screenshots and runtime state, not only with unit tests.
